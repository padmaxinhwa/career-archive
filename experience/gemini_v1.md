# [경력기술서] 이커머스 백엔드 엔지니어 (전시 도메인)

> **"복잡한 레거시를 단순한 MSA 구조로 재설계하고, 데이터 정합성과 운영 자동화를 시스템으로 해결하는 개발자"**

---

## 🚀 Professional Summary
* **전시 도메인 특화:** 기획전, 브랜드관 등 대규모 트래픽이 집중되는 전시 서비스의 MSA 전환 및 구조 통합 주도.
* **성능 및 구조 최적화:** Java 21 Virtual Thread 도입을 통해 Reactive Stack의 복잡성을 해소하고 유지보수성이 높은 동시성 모델 구축.
* **아키텍처 거버넌스:** ArchUnit을 활용한 의존성 규칙 자동 검증 체계를 도입하여 코드 품질 유지 비용 절감.
* **운영 자동화:** 반복적인 디버깅 리소스를 줄이기 위한 ‘상품 유효성 검증 도구’를 직접 설계/개발하여 운영 효율 획기적 개선.

## 💻 Key Projects

### 1. Java 21 Virtual Thread 기반 동시성 모델 전환 (2024.xx - 2024.xx)
* **배경:** WebFlux의 비동기 이벤트 루프 모델이 주는 높은 학습 곡선과 디버깅/에러 트래킹의 어려움을 해결하고자 함.
* **수행 내용:** * 기존 WebFlux 기반의 비동기 로직을 **Java 21 Virtual Thread** 기반의 블로킹 모델로 전환.
    * 복잡한 연쇄 호출(Chaining) 구조를 명령형 코드로 단순화하여 가독성 및 에러 트래킹 효율 향상.
    * 전환 과정에서 저사용/고비용 레거시 기능을 과감히 정리하여 시스템 경량화 및 성능 저하 요소 제거.
* **성과:** 개발 복잡도 감소 및 유지보수 생산성 향상, 장애 발생 시 스택 트레이스 가시성 확보.

### 2. 공식브랜드관 인덱스 MSA 전환 및 데이터 동기화 설계 (2023.xx - 2024.xx)
* **배경:** 레거시 인덱스 구조의 중복 데이터 타입과 강한 결합도로 인한 확장성 한계 및 운영 비효율 발생.
* **수행 내용:** * **MongoDB 인덱스 컬렉션 재설계:** 데이터 타입 통합(2종 → 1종) 및 확장 가능한 스키마 설계.
    * **동기화 파이프라인 구축:** 일배치(전체 동기화) + Kafka Consumer(실시간 BO 변경분 반영) 구조 구현.
    * Kotlin을 적용하여 도메인 로직의 간결함 확보 및 레거시 구조 통합.
* **성과:** 대카테고리/모듈매장 등 주요 영역의 MSA 전환 완료 및 신규 구좌 확장 속도 개선.

### 3. 기획전 상품 조회 API Chain 고도화 및 정합성 개선 (2024.xx - 2025.xx)
* **배경:** DB 직접 조회 방식에서 발생하는 품절 상품 노출 및 재고 정합성 이슈 해결 필요.
* **수행 내용:** * **API Chain 설계:** [Filter API(재고/노출) → Unit API(상품정보) → Post Validation(정합성 재검증)] 흐름 표준화.
    * 5개 이상의 API를 재설계하여 품절 상품 하단 정렬 및 전시 미노출 예외 처리 로직 강화.
    * **등록 구조 정규화:** 1 Row = 1 상품 구조로 개선하여 운영 휴먼에러 및 수기 등록 부담 감소.
* **성과:** 상품 노출 관련 이슈 감소 및 정합성 개선, 전시 정책 코드 기반의 관리 체계 정립.

### 4. 운영 자동화: 상품 유효성 검증기(Internal Tool) 개발
* **배경:** MSA 전환 후 복잡해진 서비스 간 호출 흐름으로 인해 운영 문의 대응 시 수기 로그 분석 리소스 과다 발생.
* **수행 내용:** * 기획전 마스터 정보와 재고 필터 API 파라미터를 자동 생성 및 시각화하는 내부 개발자 도구 개발.
    * 특정 상품의 노출/미노출 원인을 재현하고 유효성 검사 로직을 자동 수행하는 기능 구현.
    * 기술 스택: Kotlin, Spring Boot, Bootstrap, Thymeleaf.
* **성과:** **운영 대응 프로세스 자동화**를 통한 이슈 분석 시간 단축 및 수기 디버깅 리소스 절감.

## 📈 Architecture & Governance
### ArchUnit 기반 아키텍처 검증 자동화
* **수행 내용:** 주요 패키지(store, external, bizcore) 대상 ArchUnit 테스트 작성 및 의존성 규칙 정의.
* **적용 규칙:** `store` 패키지 외부 참조 금지, `@BizComponent` 강제 적용 등 계층 간 독립성 검증.
* **성과:** 코드리뷰 기반 구조 검증을 **테스트 기반 자동 검증**으로 전환하여 구조 오염 사전 차단.

## 💬 Engineering Philosophy
* **Simplicity:** 기술은 복잡함을 자랑하기 위함이 아니라, 비즈니스 복잡도를 낮추기 위해 존재해야 합니다.
* **Systemic Solution:** 반복되는 운영 문제는 사람의 주의력이 아닌, 시스템과 도구로 해결해야 합니다.
* **Governance:** 아키텍처는 관습이 아니라 자동화된 테스트로 지켜져야 지속 가능합니다.
