# Backend Developer Career Overview

---

## 1. Profile

**백엔드 개발자 (전시 도메인 중심)**

- MSA 전환 및 구조 통합 경험
- Java 21 기반 Virtual Thread 전환 경험
- Mongo 기반 인덱스 설계 및 배치/컨슈머 개발
- ArchUnit을 통한 아키텍처 검증 자동화
- 운영 자동화 도구(상품 유효성 검증기) 개발
- 시즌성 대규모 트래픽 서비스 운영 경험

## 3. 경력 요약

전시 도메인(기획전, 공식브랜드관, 킬러몰, 명절매장)에서  
레거시 구조 → MSA 전환 → 구조 단순화 → 운영 자동화까지  
전체 라이프사이클을 경험한 백엔드 개발자입니다.

- 인덱스 MSA 전환 설계 및 배치/컨슈머 구현
- WebFlux → Virtual Thread 전환을 통한 동시성 모델 개선
- 기획전 API 체인 기반 상품 조회 고도화
- 전시 정책 코드 기반 통합
- ArchUnit을 통한 아키텍처 규칙 자동 검증 체계 구축
- 운영 디버깅 자동화를 위한 내부 개발자 도구 개발

---

## 4. 주요 프로젝트

---

### ■ 공식브랜드관 인덱스 MSA 전환

**역할**
- 설계 및 개발 담당

**수행 내용**
- Mongo 인덱스 컬렉션 설계
- 일배치 개발 (공식스토어 스코어 반영)
- Consumer 개발 (BO 변경 데이터 반영)
- Kotlin 적용
- 레거시 인덱스 구조 통합

**성과**
- 대카테고리/모듈매장/인덱스 페이지 전환 완료
- 데이터 타입 통합 (NEW_SPCSHOP 통합)
- 확장 가능한 구조 확보

---

### ■ 기획전 MSA 상품 조회 고도화

**목표**
- 재고 판단 기준 및 노출 우선순위 개선

**개선 내용**
- Mongo 직접 조회 → item-valid-webflux 호출 구조 전환
- API 체인 구조 (필터 → 유닛 → 정합성 재검증)
- 품절 상품 하단 정렬
- 전시미노출 예외 처리

**성과**
- 미노출 이슈 감소
- 정합성 개선
- API 구조 표준화

---

### ■ Java 21 전환 (WebFlux → Virtual Thread)

**수행 내용**
- WebFlux 이벤트 기반 비동기 구조 제거
- Java 21 Virtual Thread 기반 동시성 모델 전환
- 블로킹 기반 코드 작성 가능 구조 전환
- 복잡도 감소
- 에러 트래킹 용이성 향상

**추가 개선**
- 저사용 기능 제거 (구분자별 우선점포 기능 삭제)
- 성능 저하 요소 정리
- 그룹기획전 구조 유도

**성과**
- 개발 복잡도 감소
- 디버깅 용이성 향상
- 구조 단순화 및 유지보수성 개선

---

### ■ 기획전 등록 구조 개선 (2025)

**AS-IS**
- 양몰 상품 등록 시 다중 Row 생성
- 중복 및 휴먼에러 발생

**TO-BE**
- 1 Row = 1 상품 구조로 정규화
- D571 코드 기반 등록 가능 사이트 정제
- D0668 코드 기반 전시 정책 통합

**성과**
- 운영 정확도 향상
- 수기 등록 부담 감소
- 전시 정책 코드 기반 관리 체계 정립

---

### ■ 기획전 상품 유효성 검증기 (내부 개발자 도구)

**배경**
- MSA 전환 이후 재고 동기화 이슈 증가
- 수기 디버깅 리소스 과다

**수행 내용**
- 기획전 마스터/구분자 정보 제공
- 재고 필터 API 호출 파라미터 제공
- 상품 유닛 API 호출 흐름 재현
- 유효성 검사 로직 자동 수행
- 동기화 수행 기능 추가

**기술**
- Kotlin
- Spring Boot
- Bootstrap
- Thymeleaf

**성과**
- 수기 API 호출 및 로그 분석 프로세스 자동화
- 운영 문의 대응 속도 개선
- 상품 노출 이슈 원인 분석 자동화

---

### ■ ArchUnit 적용 (공식브랜드관 MSA)

**목표**
- 아키텍처 의존성 규칙 자동 검증

**수행 내용**
- Core API 기반 ArchUnit 테스트 작성
- store / external / bizcore 패키지 의존성 규칙 정의
- @BizComponent 강제 적용
- 패키지 간 독립성 검증

**규칙**
- store 패키지 외부 참조 금지
- @BizComponent 직접 의존 금지
- bizcore 패키지 간 상호 의존 제한

**성과**
- 아키텍처 규칙 테스트 자동화
- 코드리뷰 기반 구조 검증 → 테스트 기반 자동 검증 전환
- 구조 오염 사전 차단

---

### ■ 명절 어드민 및 배치 MSA 전환

- 프로시저 기반 명절 상품 배치 → MSA 배치 전환
- BO 페이지 개발 (가격대/N+1/기간 관리)
- 마케팅 직접 운영 가능 구조 설계

---

### ■ 퀵커머스 어드민 개발

- 별도 BO/PO 구조 설계
- 초기 세팅 구조 협의 및 구현
- 신규 버티컬 도메인 확장 경험
