# [Senior Check-list] 경력기술서 고도화 가이드

시니어 서버 개발자로서의 전문성과 설득력을 높이기 위해 보완해야 할 핵심 포인트 4가지와 예상 질문 대응 전략입니다.

---

## 1. "So What?" - 비즈니스 임팩트의 구체화
단순한 기능 구현(Output)이 아닌, 비즈니스에 기여한 실질적 가치(Outcome)를 수치로 증명해야 합니다.

* **부족한 점:** "성능 개선", "이슈 감소" 등 주관적이고 추상적인 표현
* **보완법 (예시):**
    * **처리량:** "Virtual Thread 전환 후 기존 대비 CPU 사용률 00% 절감, 동일 인프라 사양 기준 TPS 00% 향상"
    * **비용/시간:** "운영 자동화 도구 도입 후, 평균 30분 소요되던 장애 원인 파악 리소스를 5분 이내로 단축 (80% 개선)"
    * **안정성:** "명절/시즌 이벤트 시 발생하는 평시 대비 10배 이상의 트래픽에도 무정지 서비스 서빙"

---

## 2. "Trade-off" - 기술 선택의 근거와 고민
시니어는 기술의 '장점'만 보지 않습니다. 도입 시의 리스크와 기회비용을 어떻게 관리했는지가 핵심입니다.

* **부족한 점:** WebFlux에서 Virtual Thread로 전환할 때의 기술적 리스크 관리 과정 생략
* **보완법 (예시):**
    * **전환 전략:** "전체 시스템 리스크를 최소화하기 위해 신규 API부터 단계적으로 적용하는 Strangler Fig Pattern 채택"
    * **문제 해결:** "Virtual Thread 도입 시 발생할 수 있는 Pinning 현상(synchronized 블록 이슈)을 파악하고, ReentrantLock 전환을 통해 동시성 저하 방지"

---

## 3. "Data Consistency" - 분산 환경 정합성 해결
이커머스 전시 도메인의 핵심은 "데이터 정합성"입니다. 시스템 장애나 지연 상황에서의 대응책을 보여주어야 합니다.

* **부족한 점:** 배치 및 Consumer를 통한 동기화의 해피 케이스만 언급 (실패 시나리오 부재)
* **보완법 (예시):**
    * **메시징 보장:** "Idempotent Consumer 설계를 통해 Kafka 메시지 중복 처리 방지 및 Transactional Outbox Pattern 검토"
    * **장애 복구:** "배치 실패 시 자동 재처리(Retry) 전략 및 데이터 불일치 감지를 위한 정기 정정 프로세스(Reconciliation) 구축"

---

## 4. "Leadership & Mentoring" - 조직적 기여
개인의 성과를 넘어 팀 전체의 생산성과 기술 수준을 어떻게 끌어올렸는지 기술합니다.

* **부족한 점:** 개인의 개발 성과에 치중된 서술
* **보완법 (예시):**
    * **지식 전파:** "Java 21 도입 및 가상 스레드 활용 가이드라인을 사내 위키에 전파하고 기술 세미나 주도"
    * **품질 표준:** "ArchUnit 도입으로 코드 리뷰 시 아키텍처 위반 확인 리소스를 00% 단축하고 팀 내 구조적 일관성 확보"
    * **자산 최적화:** "기술 부채 청산(미사용 모듈 제거)을 통해 전체 프로젝트 빌드 속도 00% 개선 및 운영 인프라 비용 효율화"

---

## 💡 시니어 면접관의 '압박 질문' 대비
경력기술서 행간에 아래 답변이 녹아있다면 전문성이 배가됩니다.

* **Q. Virtual Thread 전환 후 힙 메모리(Heap Memory) 이슈는 없었나요?**
    * *(준비)* 가상 스레드 수가 급증할 때 각 스레드의 Stack 할당으로 인한 메모리 압박과 이를 모니터링/제한한 경험.
* **Q. MongoDB 인덱스 설계 시 읽기 성능 최적화를 위해 무엇을 고려했나요?**
    * *(준비)* 전시 영역의 쿼리 패턴을 분석하여 Compound Index의 필드 순서 최적화 또는 Partial Index를 활용한 인덱스 크기 효율화 사례.
* **Q. API Chaining 구조의 고질적인 문제인 네트워크 레이턴시 누적은 어떻게 해결했나요?**
    * *(준비)* 병렬 호출(Parallel Call) 처리, 타임아웃 전략 수립, 그리고 서킷 브레이커를 통한 Fallback 로직 적용 경험.
